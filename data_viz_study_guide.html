📈 Chart Types & Usage

Line Graph

Trends Over Time

Best for: Showing changes over time, continuous data

Key feature: Zero baseline NOT required

Bar Chart

Comparisons

Best for: Comparing categories or values

Limitation: Comparing segments at baseline is difficult (stacked bars)

Slope Graph

Before/After

Best for: Visualizing before-and-after comparisons

Example: Performance metrics pre- and post-policy implementation

Waterfall Chart

Cumulative Change

Best for: Showing how a starting value changes through positive/negative contributions

Example: Budget breakdown, profit/loss analysis

Pie Charts - When Acceptable?

According to some practitioners, pie charts are acceptable when:

  • The pie chart has very few slices (minimal categories)
  • You're NOT comparing many pie charts side-by-side
  • The slices do not need to add up to exactly 100%
  • Categories/slices are NOT very close in size
⚠️ Caution: Pie charts are often criticized because comparing angles/areas is difficult for the human visual system.

📊 Chart Selection Examples

Q: Show profit over time?
A: Line chart (best practice: most recent date on right, going left to right)

Q: Show monthly revenue with breakdown of income vs. expenses?
A: Waterfall chart

Q: Compare Q1 vs. Q2 sales across departments?
A: Slope graph

🎨 Gestalt Principles

Gestalt principles describe how humans naturally perceive and group visual elements. Understanding these helps create more effective visualizations.

Proximity

Objects that are close together are perceived as a group.

Application: Use spacing to direct readers to scan across rows or down columns by grouping related data.

Similarity

Objects that share visual characteristics (color, shape, size) are perceived as related.

Application: Use consistent colors for related data categories.

Enclosure

Objects within a boundary are perceived as a group.

Application: Use boxes or shading to group related information.

Closure

We mentally "close" gaps to perceive complete shapes.

Application: Define areas using x and y axes without needing all four borders.

Continuity

Our eyes naturally follow lines and curves.

Application: Arrange elements along a path to guide attention.

Connection

Objects that are physically connected are perceived as related.

Application: Line graphs connecting data points show relationships over time.

Strength Order: Connection is one of the STRONGEST grouping principles (Continuity → Enclosure → Connection is strongest)

👁️ Preattentive Attributes

Preattentive attributes are visual properties processed rapidly and automatically by the human visual system, requiring minimal cognitive effort.

What is a Preattentive Attribute?

A visual property that is processed rapidly and automatically, typically by the human visual system. These attributes help viewers quickly decode visualizations.

Key characteristic: They allow viewers to detect patterns without conscious thought.

Common Preattentive Attributes:

  • Position - Where something is located (most effective for quantitative comparison)
  • Length - Size/length of bars, lines
  • Color - Hue, saturation, brightness
  • Shape - Different symbols or forms
⚠️ Important: Position is NOT a preattentive attribute according to the exam definition. Shape is the weaker attribute for quantitative comparison - shapes are difficult to compare accurately.

🧠 Cognitive Load

Definition

Cognitive load refers to the mental effort required to process information. In data visualization, high cognitive load means viewers must work harder to understand the chart.

Reducing Cognitive Load

High cognitive load forces viewers to:

  • Spend more effort decoding the chart instead of interpreting the data
  • Makes visualizations less effective for communication

To reduce cognitive load:

  • Remove unnecessary elements (declutter)
  • Use preattentive attributes effectively
  • Eliminate heavy borders and chart frames
  • Use clear axis titles and labels

Examples:

Increases cognitive load: Heavy borders, dense gridlines, unnecessary 3D effects, too many colors

Decreases cognitive load: Direct labels, minimal gridlines, bold color for key data, simple design

🔍 Clutter

Definition

Clutter refers to visual elements that take up space but do not increase understanding. These elements distract from the data story.

Examples of Clutter:

  • Unnecessary gridlines
  • Heavy chart borders and frames
  • Excessive data markers
  • Decorative elements without purpose
  • Too many colors
  • Redundant legends when direct labels work better
💡 Best Practice: Visual elements should take up space but DO increase understanding. If it doesn't add value, remove it!

📊 Tableau Fundamentals

Dimensions vs. Measures

Dimensions:

  • Typically answer "how much" or "how many"
  • Qualitative/categorical data
  • Examples: Customer Name, Category, Date, Product Name
  • Used to slice and group data

Measures:

  • Quantitative numerical values
  • Examples: Sales, Profit, Quantity, Price
  • Can be aggregated (sum, average, count, etc.)
  • Typically used on axes to show amounts

✨ Visualization Best Practices

The Interpretation Gap

The difference between what the data shows and what audiences understand without additional narrative or context.

Solution: Provide storytelling, annotations, and context to bridge this gap.

Storytelling with Data

Storytelling provides context and meaning to raw numbers. It helps audiences:

  • Understand the "why" behind the data
  • Connect emotionally with insights
  • Remember key takeaways
  • Make informed decisions

NOT primarily for: Complexity, more variables, or statistical significance alone

Exploratory vs. Explanatory Visualizations

Exploratory:

  • Analyzes data for understanding
  • Gives users tools to find their own answers
  • Interactive and flexible
  • Does NOT bypass the discovery process

Explanatory:

  • Communicates a specific story or insight
  • Guides audience to a conclusion
  • More static and focused

🎨 Color Usage

Color Best Practices

  • Use it sparingly: Too many colors create confusion
  • Use it consistently: Same color = same meaning across visualizations
  • Design with color-blind in mind: Use colorblind-friendly palettes
  • Be thoughtful of tone: Colors convey emotion and meaning (red = negative, green = positive in Western contexts)
⚠️ Risk of Enclosure: Overusing enclosure (boxes, borders) can create unnecessary visual weight and clutter.

📝 Text in Visualizations

When to Use Text Alone

Use text instead of visualizations when:

  • You have a single number or simple comparison
  • The message is straightforward
  • A chart would be unnecessary complexity

Example: "Sales increased 15% this quarter" is clearer as text than a simple bar chart.

💡 Remember: Not every data point needs a visualization. Sometimes simple text is the most effective communication method.

📊 Other Design Principles

Tables

Why avoid in presentations?

  • Audience reads them instead of listening to you
  • They lose their attention
  • Don't display enough information (compared to charts)
  • Are not professional enough

✓ Correct answer: As the audience reads them, you lose their attention

Gridlines

Should visualizations have gridlines?

False - Visualizations should NEVER have gridlines as a blanket rule.

Nuance: Gridlines can be helpful in some contexts but should be used minimally and only when they improve understanding.

Visual Hierarchy

Without other visual cues, audiences typically start at the top left of a visualization (Western reading convention).

Lack of Clear Contrast

Lack of clear contrast can be a form of visual clutter because it makes the visualization harder to read and process.

📐 Tidy Data Principles

Tidy Data Format

In tidy data principles, each variable should be in one column. This makes data:

  • Easier to analyze
  • Easier to visualize
  • More consistent and organized
⚠️ Common Mistakes:
  • Having variables in multiple columns (for redundancy) - WRONG
  • Aggregating data before storage - depends on use case
  • Converting to text format - not a tidy data principle

🔄 The Visualization Process

Is Visualization a One-Time Activity?

TRUE: Visualization is rarely a one-and-done activity. It's typically an iterative process involving:

  • Initial creation
  • Review and feedback
  • Refinement and adjustment
  • Testing with audiences
  • Final polish
